# Проект: Таймер тренировки — Дзюдо Самара

## Контекст
Интервальный таймер для тренировок дзюдо. Отдельный инструмент от сайта федерации ([[judo-samara]]) — общий бренд/палитра, но разные задачи и разный жизненный цикл.

## Статус
Реализован по SPEC.md (см. в этой же папке). Два экрана: программирование тренировки и рабочий экран с крупным отсчётом, голосовыми командами (Хаджимэ/Матэ/Приготовились/Соромадэ) и цветовой индикацией фазы. PWA, работает офлайн.

Приложение называется **judo-timer** (title/manifest/брендинг на экране настройки) — переименовано 2026-07-21, до этого не имело отдельного имени от сайта федерации.

На экране настройки нет встроенных пресетов-вкладок (Рандори/Учикоми/Табата убраны 2026-07-21) — просто список полей + отдельный сворачиваемый блок «Сохранённые тренировки» для тренерских заготовок, сохранённых под своим названием.

2026-07-21: у каждого поля настройки — своя inline-SVG иконка (без внешних шрифтов/иконок, офлайн-совместимо), шрифты переведены на системный стек (system-ui — без внешних загрузок), убран uppercase с фазовых подписей на рабочем экране (были заглавными только из-за CSS, в PHASE_LABELS текст всегда был обычным регистром).

2026-07-21 (по итогам первой обкатки тренером на телефоне): убрана фаза «Заминка» (cooldown) целиком — из настроек, логики и рабочего экрана; тренировка теперь заканчивается «Соромадэ» сразу после последней «Работы». Бип-отсчёт унифицирован до последних 3 сек любой фазы (раньше в подготовке был запас 10 сек). Разблокировка звука (`unlockAudio()`) теперь асинхронно дожидается всех аудио-элементов перед стартом отсчёта (с таймаутом 1.2с) — на самом первом холодном запуске «Приготовились» могло проглатываться из-за гонки. Возврат к настройкам: кнопки поменяны местами по просьбе тренера — «Старт» теперь всегда основное действие и начинает новую тренировку (даже если есть тренировка на паузе — она перезаписывается), а «Продолжить предыдущую тренировку» стало мелкой отдельной ссылкой, показывается только когда есть пауза. Само продолжение (если им воспользоваться) больше не сбрасывает прогресс внутри текущей фазы на её начало и не переозвучивает её заново.

## Структура
- `SPEC.md` — согласованная с тренером спецификация (интервью, краевые случаи)
- `index.html`, `styles.css`, `js/*.js` — модульная реализация
- `assets/audio/` — голосовые фразы (записаны тренером)
- `manifest.webmanifest`, `sw.js` — PWA/офлайн

## Открытые вопросы
