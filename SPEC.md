# Спецификация: Таймер тренировки «Дзюдо Самара» (v2)

Статус: черновик после интервью с заказчиком, 2026-07-18. Реализация ещё не начата — это ТЗ для последующей разработки.

## 1. Идея, которую сохраняем

Единственное, что жёстко фиксировано и не подлежит пересмотру: **интервальный таймер тренировки с голосовыми командами «Хаджимэ» (начало работы) и «Матэ» (стоп/отдых)**, звучащими автоматически по ходу отсчёта. Всё остальное — структура кода, UI, набор экранов — меняется радикально относительно текущей версии (`index.html`, 1.8 МБ, всё в одном файле).

## 2. Кто и как пользуется

- Тренер один программирует тренировку и запускает её на **одном устройстве** — универсально: Android, iPhone или что окажется под рукой (телефон/ноутбук/Smart TV браузер). Специальной оптимизации под конкретную платформу не делаем, но проверяем, что работает и на Android, и на iOS Safari (там строже с автовоспроизведением звука и с тем, чтобы экран не гас).
- Во время работы экран этого же устройства виден **издалека всему залу** (в руке, на подставке или через каст на TV/проектор) — рабочий экран должен читаться на расстоянии.
- Нет отдельного «пульта» и отдельной «витрины» — это один и тот же экран одного и того же устройства, просто до старта на нём конструктор, а после старта — табло.

## 3. Структура приложения: два экрана

Технически — одна страница (SPA), но с двумя явно разделёнными видами, переключаемыми без перезагрузки (чтобы не терять состояние таймера в памяти):

### Экран 1 — «Программирование»
Здесь тренер настраивает план тренировки перед стартом:
- Подготовка (prep), Работа (work), Отдых (rest), Циклы, Сеты, Отдых между сетами (restSets) — сохраняем текущую двухуровневую модель «сеты из циклов». *(Изменено 2026-07-21: поле «Заминка» (cooldown) убрано целиком по итогам обкатки — тренировка заканчивается сразу после последней «Работы».)*
- Никаких готовых пресетов-вкладок (Рандори / Учикоми / Табата) — экран программирования это просто список полей для ручной настройки, без переключателей сверху. *(Изменено 2026-07-21: раньше здесь были встроенные пресеты-«вкладки», от них отказались в пользу простого списка полей + отдельного режима «Сохранённые тренировки» ниже.)*
- Тренер может **сохранить свою комбинацию параметров под собственным названием** — кнопка «Сохранить эту тренировку». Сохранённые тренировки открываются отдельным списком по кнопке «Сохранённые тренировки» (свёрнут по умолчанию, чтобы не загромождать экран), каждая — с возможностью выбрать (подставить её параметры в поля) или удалить.
- Значения времени задаются **и кнопками +/- (для мелкой подстройки), и прямым вводом цифр** (напечатать точное число секунд/минут), без необходимости листать +/- десятки раз.
- Последние введённые значения и все сохранённые тренировки **запоминаются автоматически** между запусками приложения (localStorage) — начинать с нуля каждый раз не нужно.

### Экран 2 — «Рабочий экран»
Включается по кнопке «Старт». Показывает:
- Крупный обратный отсчёт текущей фазы.
- **Цвет всего фона/экрана меняется в зависимости от фазы** (подготовка/работа/отдых) — это основной сигнал, различимый издалека, без необходимости всматриваться в текст. Текстовые подписи фазы (например «Работа», «Отдых») остаются как вторичный, некрупный элемент для доступности — крупно дублировать «ХАДЖИМЭ»/«МАТЭ» на экране не нужно, это и так слышно голосом.
- Счётчик текущего цикла/сета.
- **Общее оставшееся время всей тренировки** (не только текущей фазы).
- **Кнопка паузы** — тренер может в любой момент остановить отсчёт и продолжить с того же места.
- **Переход назад к экрану программирования без сброса прогресса** — тренировка на паузе не теряется, но по возврату основное действие кнопки «Старт» — **начать новую тренировку** с текущими настройками (перезаписывает то, что было на паузе). Продолжить именно ту, прежнюю, тренировку — отдельная мелкая ссылка «Продолжить предыдущую тренировку» под кнопкой «Старт», показывается только когда есть, что продолжать. *(Уточнено 2026-07-21: раньше было наоборот — «Продолжить» было основным действием, а «Начать заново» мелкой кнопкой; тренер попросил поменять местами.)* Само продолжение (если им воспользоваться) идёт с той же секунды, на которой остановились, без повторного объявления фазы голосом.
- **Стрелки «назад»/«вперёд» по циклам** — переход на соседний цикл за одно нажатие (несколько нажатий подряд = переход через несколько циклов). Полноценный «прыжок» на произвольный номер реализуется через повторные нажатия стрелки, отдельного списка/ввода номера не делаем.
- Кнопки выключения звука на рабочем экране **нет** — звук на этом экране всегда включён и работает.

**Пересчёт общего времени при переходе по циклам («вариант А»):** если тренер перескакивает вперёд/назад по циклам, «Осталось» пересчитывается честно — под пропущенные фазы время из общего остатка вычитается (или добавляется при шаге назад), чтобы цифра всегда отражала реальный оставшийся план, а не исходный.

## 4. Голосовые команды и звук

Приоритет — заранее записанные аудиофайлы (как в текущей версии), с fallback на Speech Synthesis API браузера, если файл не воспроизвёлся.

Фразы:
| Фаза | Фраза | Когда звучит |
|---|---|---|
| Подготовка (prep) | «Приготовились» | начало каждой фазы подготовки |
| Работа (work) | «Хаджимэ» | начало каждого рабочего цикла |
| Отдых (rest / restSets) | «Матэ» | начало каждого отдыха (в т.ч. между сетами) |
| Окончание всей тренировки (done) | «Соромадэ» | один раз, когда вся тренировка полностью завершена |

«Приготовились» и «Соромадэ» — новые фразы, отсутствующие в текущей версии (сейчас на эти события просто пищит бип / читает нейтральный TTS-текст). Их нужно записать/добавить как аудиофайлы аналогично существующим «Хаджимэ»/«Матэ».

Короткий бип-отсчёт (не голос) звучит в последние **3 секунды** любой фазы, без исключений. *(Уточнено 2026-07-21: раньше в подготовке бип начинался за 10 сек до конца — унифицировано до 3 сек везде; голосовое «Приготовились» для длинной подготовки по-прежнему звучит заранее, это отдельный сигнал.)* Без «Заминки» тренировка теперь заканчивается сразу после последней «Работы» — сразу «Соромадэ», без «Матэ» между ними.

Аудиофайлы (голос заказчика), уже лежат в `assets/audio/`:
- `hajime-user.wav`, `mate-user.wav` — готовы из старой версии.
- `soromade-user.oga` — прислан заказчиком через Telegram 2026-07-18, звучит «Соромадэ», используется на событие «done» (конец всей тренировки). Формат .oga, при реализации привести к общему формату с остальными (.wav) или подтвердить, что .oga воспроизводится штатно.
- `prigotovilis-user.oga` — прислан заказчиком через Telegram 2026-07-18, «Приготовились» (используется на событие «prep», начало подготовки). Подтверждено.

> ⚠️ Отдельно зафиксировано пожелание заказчика: после реализации потребуется **коррекция звука** (громкость/тайминг/сведение голосовых файлов) — отдельный проход уже после того, как таймер заработает функционально. Не забыть на финальном этапе.

## 5. Конец тренировки

Когда прозвучало «Соромадэ» (все сеты/циклы/заминка пройдены) — приложение **автоматически возвращается на экран программирования**, без ручных действий тренера.

## 6. Технические требования

- **Экран не должен гаснуть/блокироваться** во время работы таймера (Wake Lock API), обязательно.
- **Таймер должен считать по реальному времени (timestamp), а не просто декрементом раз в секунду** — если браузер на секунду-другую придержит `setInterval` (сворачивание, фоновый режим, слабое устройство), отсчёт всё равно должен доехать до правильного значения при возврате в фокус, без накопленной ошибки. Это меняется относительно старой версии, где `tick()` просто вычитал 1 каждую секунду.
- Устойчивость к случайной перезагрузке страницы: текущее состояние тренировки (фаза, время, номер цикла/сета) стоит сохранять в localStorage/sessionStorage на лету, чтобы можно было восстановиться, если вкладка случайно перезагрузилась посреди тренировки.
- Работает офлайн как раньше (service worker / PWA), устанавливаемо на телефон.
- Стек: обычный HTML/CSS/JS без сборки, но **разбитый на отдельные файлы** (разметка / стили / логика setup-экрана / логика run-экрана / аудио-модуль), а не один файл на 1.8 МБ — вместо монолита нужна понятная модульная структура, проще поддерживать и расширять.

## 7. Краевые случаи

- Нулевая длительность необязательной фазы (rest = 0, restSets = 0, сеты = 1) — фаза целиком пропускается, без объявления и без паузы на 0 секунд.
- Стрелки перехода по циклу на границах (первый/последний цикл) — не уводят за пределы тренировки: «вперёд» с последнего рабочего цикла естественно продолжает обычный сценарий (отдых → заминка → конец), «назад» с первого цикла — no-op.
- Двойное быстрое нажатие «Старт/Пауза» подряд — не должно ломать состояние (дебаунс).
- Поворот экрана телефона (портрет/альбом) во время тренировки — layout адаптируется, отсчёт не сбрасывается.
- Каст на TV с телефона: Wake Lock держит только экран телефона-источника — если экран телефона всё же погаснет, трансляция на TV может прерваться. Ограничение платформы, стоит держать это в уме (в UI можно ненавязчиво напомнить не выключать экран телефона при касте).
- Отсутствие/ошибка загрузки аудиофайла — TTS-fallback должен сработать надёжно, тренировка не должна "молчать".
- Первый тап по «Старт» разблокирует аудио (autoplay policy) — сохранить логику `unlockAudio()` из старой версии.
- Минимальные значения параметров (например, work не может быть 0) — сохранить разумные минимумы, аналогичные текущим (`ADJ_MIN`).

## 8. Не в этой версии / открытые вопросы на будущее

- Коррекция звука (см. раздел 4) — делается отдельным проходом позже.
- Возможность синхронизации «пульт + отдельная витрина» на двух устройствах не нужна — явно исключено сценарием использования.
- Кнопка «выключить звук» на рабочем экране не нужна — явно исключено.
